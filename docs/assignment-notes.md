# Assignment Notes

## 이 프로젝트에서 지킨 원칙
- 과제 MVP 외 기능은 넣지 않는다.
- 입력 검증 규칙과 에러 포맷을 고정해서 예측 가능한 동작만 제공한다.
- 화면은 바닐라 JS로 단순하게 구성한다.

## 서버 규칙 요약
- 커스텀 확장자 정규화: `trim -> lower-case`
- 허용 패턴: `^[a-z0-9]{1,20}$`
- 고정 확장자와 커스텀 이름 충돌 금지
- 커스텀 중복 금지
- 커스텀 최대 200개

## 추가 고려사항 (이번 구현 범위 밖)
- 실제 파일 업로드 시 확장자 차단 연결
- 사용자/조직 단위 정책 분리
- 변경 이력(audit) 저장
- 운영 모니터링/알림
